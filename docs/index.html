01 — Pipeline

4 stages from question
to insight

Every query flows through a deterministic 4-stage pipeline. Input guardrails run first, then context is assembled with the user's financial profile, the LLM reasons over it, and output guardrails verify the response before delivery.

1
Input & Validation
Unicode normalization, injection detection, scope check, cross-user leakage prevention, follow-up resolution
2
Context Assembly
User profile from cache, conversation history (6-msg window), query history, system prompt with financial data
3
LLM + Tool Dispatch
31 intent handlers, LLM reasoning with 12 tool schemas, chart generation via Plotly, smart fallback chain
4
Response & Guard
XSS sanitization, hallucination grounding, toxicity filter, cache update, audit logging
02 — Features

More than a chatbot.
Financial intelligence.

💬

Proactive Conversations

Every response ends with a contextual follow-up question. Reply with yes/no, a number, or just a topic word like "groceries" — the AI understands and continues.

📊

12 Chart Types

Donut breakdowns, trend lines, heatmaps, scatter plots, cumulative flow, merchant deep-dives — all rendered server-side with Plotly in a premium dark theme.

💰

Smart Budget Planning

Prescriptive budget tables with the 50/30/20 rule. Multi-month plans (quarter, 6 months) with progressive reduction targets and projected savings.

🔄

Subscription Detection

Automatically identifies recurring charges from transaction patterns. Classifies subscriptions vs bills, calculates total monthly outflow, suggests cancellation targets.

🕐

Dynamic Time Periods

"Last 3 months", "since January", "this quarter" — the pipeline parses natural language time references and filters data accordingly.

🧠

Multi-Turn Memory

6-message sliding window conversation history. The AI remembers what you asked, what charts it showed, and builds on previous context.

📝

Comprehensive Reports

Request a "full financial report" and get a multi-section analysis with 8+ charts covering spending, income, savings, trends, and merchant breakdowns.

🔀

LLM Fallback Chain

8 models across 2 providers (Sarvam AI + OpenRouter). If one model fails, the next picks up seamlessly — zero downtime for the user.

04 — Security

12 guardrails, zero compromises

Production-grade security across input, output, and operational layers. Every request passes through all checks before reaching the LLM.

🔤

Unicode Normalization

NFKC normalization + zero-width char stripping defeats homoglyph bypass attacks

🛡️

Prompt Injection Detection

16 regex patterns catch "ignore instructions", jailbreak, DAN mode, role hijacking

🔒

Cross-User Leakage Prevention

Blocks any reference to other user IDs or "another user" patterns

🎯

Scope Validation

100+ financial keywords + conversational whitelist ensure on-topic queries

✂️

Input Length Truncation

Hard cap at 500 chars prevents oversized prompts from abusing context

🧹

XSS Sanitization

14 patterns strip <script>, <iframe>, on*= handlers, javascript: URIs from output

📐

Hallucination Grounding

Every dollar amount in LLM output is validated against actual data (10% tolerance)

🤔

Confidence Detection

Flags "I'm not sure" and "insufficient data" responses for review

🚫

Toxicity Filter

Keyword-based filter catches offensive, violent, or inappropriate LLM output

⚡

Circuit Breaker

Trips after 5 consecutive LLM failures, auto-resets after 60 seconds

📋

Audit Logging

Hashed user IDs, truncated prompts, latency, flags — every request is logged

⏱️

Rate Limiting

10 req/min per user, 60 req/min global — sliding window, in-memory

06 — Quick Start

Up and running
in 3 steps

# 1. Clone and install
$ git clone https://github.com/pratiksinha312511/Tabular_RAG_pipeline.git
$ cd Tabular_RAG_pipeline
$ python -m venv .venv && .venv\Scripts\activate
$ pip install -r requirements.txt

# 2. Configure environment
$ cp .env.example .env
# Edit .env — add your SARVAM_API_KEY or OPENROUTER_API_KEY

# 3. Run
$ python server.py
# → Open http://localhost:8000

Environment Variables

Variable Description
SARVAM_API_KEY Sarvam AI API key (primary LLM provider, BYOK)
OPENROUTER_API_KEY OpenRouter API key (fallback provider, free models)
CSV_FILE Transaction CSV filename in data/ folder
LLM_PROVIDER Force provider: sarvam or openrouter (default: auto-detect)
PORT Server port (default: 8000, auto-set by Render/Railway)
07 — Deploy

One-click deploy
to Render.com

1️⃣

Connect GitHub

Go to render.com → New Web Service → connect your GitHub repo. Render auto-detects the Dockerfile.

2️⃣

Set Environment

Add SARVAM_API_KEY, OPENROUTER_API_KEY, and CSV_FILE in the Environment tab. Render handles PORT automatically.

3️⃣

Deploy

Click "Create Web Service". Docker builds in ~3 min, health check runs on /api/users, and your app is live.

# Or deploy with Docker locally
$ docker build -t vola-ai .
$ docker run -p 8000:8000 --env-file .env vola-ai
# → Open http://localhost:8000
